❯ 経路計算って何キロで走った時の計算？ 

● 固定の速度ではなく、Google Maps Directions API に任せています。

  destinations.js の該当部分:

  const request = {
    origin: ...,
    destination: ...,
    travelMode: 'DRIVING',  // ← 車モードのみ指定
  };
  // leg.duration.value（秒）をそのまま使用

  つまり Google Maps が道路の種類（高速・国道・道道）・法定速度・一般的な交通状
  況をもとに算出した時間をそのまま表示しています。

  ただし drivingOptions や departureTime を指定していないため、リアルタイムの渋
  滞は反映されておらず、「平常時の所要時間」の見積もりになります。

  北海道の道の駅巡りなら実態に近いと思いますが、フェリーや季節通行止め（冬季閉鎖
  の峠など）は当然考慮されないので注意が必要です。

❯ この経路をGoogleマップに連携することは難しい？ 

● 全然難しくないです！Google Maps の URL 
  スキームを使えばリンク1つで実現できます。

  仕組み

  Google Maps はこの形式の URL で経路を開けます：

  https://www.google.com/maps/dir/出発地の緯度,経度/目的地1の緯度,経度/目的地2の
  緯度,経度/...

  例えば経路が「出発地 → A → B → C」なら：
  https://www.google.com/maps/dir/43.123,141.456/43.200,141.500/43.300,141.600/4
  3.400,141.700

  このアプリへの実装イメージ

  destinations テーブルには departure_latitude/longitude と
  destination_latitude/longitude が全部揃っているので、itineraries/show.html.erb
   などに「Googleマップで開く」ボタンを置くだけで作れます。

  制限事項

  ┌────────────┬────────────────────────────────┐
  │    項目    │              内容              │
  ├────────────┼────────────────────────────────┤
  │ 最大経由地 │ 10地点まで（出発＋目的地含む） │
  ├────────────┼────────────────────────────────┤
  │ モバイル   │ Googleマップアプリが開く       │
  ├────────────┼────────────────────────────────┤
  │ PC         │ ブラウザ版が開く               │
  └────────────┴────────────────────────────────┘

  道の駅スタンプラリーは1日10か所以上回ることもあるので、10地点制限だけ注意が必
  要です。超える場合は「1日目分だけ開く」といった日ごとの分割で対応できます。

❯ スマホのGoogleマップアプリに連携したいんだけど 

● スマホのGoogleマップアプリで開くには Google Maps URLs API 形式を使います。

  URL の形式

  https://www.google.com/maps/dir/?api=1
    &origin=出発地の緯度,経度
    &destination=最終目的地の緯度,経度
    &waypoints=経由地1の緯度,経度|経由地2の緯度,経度|...
    &travelmode=driving

  - スマホでこのURLを開くとGoogleマップアプリが起動する
  - PCでは通常のブラウザ版が開く

  制限

  - 経由地は最大8か所（出発＋最終目的地を合わせて10地点まで）
  - 1日の行程が9か所以上になると全部は入らない

  実装イメージ

  itineraries/show.html.erb の各日タブに「Googleマップで開く」ボタンを置き、その
  日の目的地データからURLを組み立てる形が自然だと思います。
